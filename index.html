Capturar — leer lo que pasa en la mesa

Herramientas para acceder a la cámara
y extraer información del mundo físico.
p5.js — createCapture(VIDEO)↗

Una línea de código para abrir la cámara en el navegador. El punto de entrada más directo: obtenés el feed sin instalar nada. Todo corre en el browser.

pedile a Claude o ChatGPT

"Dame código p5.js completo para abrir la cámara, mostrar el feed en pantalla y calcular la diferencia entre frames consecutivos para detectar movimiento."

detección de movimiento
MediaPipe — Object Detection↗

Detecta y localiza objetos en el video en tiempo real. Podés entrenar o usar modelos preexistentes para identificar los cuerpos por forma o posición.

pedile a Claude o ChatGPT

"Dame código JavaScript para usar MediaPipe Object Detector en el navegador, detectar dos objetos y calcular la distancia entre sus centros."

detección de objetos
Teachable Machine — entrenamiento visual sin código↗

Entrenás un modelo de reconocimiento directamente en el navegador con la cámara: mostrás cada cuerpo, sacás fotos de cada clase, entrenás con un clic y exportás el modelo. Sin instalar nada, sin escribir código de ML. Es el punto de partida recomendado para que el sistema reconozca los cuerpos específicos que traen al hackathon.

pedile a Claude o ChatGPT

"Tengo un modelo exportado de Teachable Machine. Dame código p5.js para cargarlo con ml5, clasificar lo que ve la cámara en tiempo real y usar la clase detectada para controlar un efecto visual."

reconocimiento de cuerpos
ml5.js — Image Classifier↗

Machine learning en el navegador sin setup. Carga modelos de Teachable Machine directamente y los integra con p5.js. El puente entre el modelo entrenado y la proyección.

pedile a Claude o ChatGPT

"Dame código p5.js con ml5 para clasificar en tiempo real lo que ve la cámara y mostrar el resultado en pantalla."

clasificación
OpenCV (Python)↗

La librería más robusta para visión computacional. Detección por color, contornos, formas. Si el equipo viene de Python, es el camino más directo y documentado.

pedile a Claude o ChatGPT

"Dame código Python con OpenCV para detectar contornos de dos objetos de colores distintos y calcular la distancia entre sus centros en tiempo real."

visión computacional
ArUco Markers↗

Marcadores visuales impresos que la cámara detecta con precisión milimétrica. Si pegás uno en cada cuerpo, el sistema sabe exactamente dónde está cada uno.

pedile a Claude o ChatGPT

"Dame código Python con OpenCV para detectar dos marcadores ArUco, obtener sus coordenadas y calcular la distancia entre ellos."

Conectar — código para unir detección y proyección

Entornos y frameworks para
construir el sistema completo.
p5.js — el entorno base recomendado↗

Cámara + detección + proyección en un solo archivo HTML. Sin instalar nada. El ecosistema más accesible para arte computacional: miles de ejemplos, integración directa con MediaPipe y ml5.

pedile a Claude o ChatGPT

"Dame el esqueleto de una sketch p5.js que: abre la cámara, detecta movimiento, y usa ese valor (0 a 1) para controlar la velocidad de reproducción de un video."

prototipo completo
TouchDesigner↗

Programación por nodos: conectás bloques sin escribir código. La cámara entra por un nodo, el movimiento detectado sale como un valor, ese valor controla efectos de video.

pedile a Claude o ChatGPT

"Explicame paso a paso cómo conectar un nodo de cámara con un nodo de velocidad de video en TouchDesigner. Soy principiante."

nodal / sin código
Max / Jitter↗

Programación visual para audio y video en vivo. Jitter procesa video frame a frame. Muy potente para manipulación sonora reactiva. Si hay alguien con experiencia, es el más flexible.

pedile a Claude o ChatGPT

"En Max/MSP, cómo uso un valor numérico para cambiar el tempo de un clip de audio en tiempo real."

audio / video live
Processing↗

Entorno clásico de arte computacional. Miles de ejemplos de instalaciones físico-digitales. Buena opción si el equipo prefiere un entorno de escritorio en lugar del navegador.

pedile a Claude o ChatGPT

"Dame código Processing que abra la cámara, detecte movimiento por diferencia de frames y use ese valor para modificar la velocidad de un video."

instalaciones
Three.js — efectos visuales reactivos↗

Si la proyección va a tener profundidad o distorsión visual, Three.js permite shaders que reaccionan a variables en tiempo real. Texturas, deformaciones, efectos de tiempo alterado.

pedile a Claude o ChatGPT

"Dame un fragment shader en GLSL para Three.js que distorsione una imagen según un valor u_encuentro entre 0 y 1."

Lenguajes — con IA para escribir el código

No hace falta saber programar de antemano.
Conocer el lenguaje le da contexto a Claude o ChatGPT para escribirlo por vos.
JavaScript

Corre en el navegador sin instalar nada. Base de p5.js, Three.js, ml5 y MediaPipe. El punto de partida más accesible: abrís un HTML y funciona.

MDN guía javascript.info p5 referencia Stack Overflow
Python

Muy usado en ciencia de datos y visión computacional. OpenCV, MediaPipe y TensorFlow tienen soporte nativo con miles de ejemplos. Ideal si el equipo viene del mundo científico.

Tutorial oficial OpenCV Python Kaggle Learn Stack Overflow
Processing (Java)

Diseñado para artistas y diseñadores. Sintaxis simple, entorno propio, miles de ejemplos de instalaciones. Históricamente el lenguaje del arte computacional.

Referencia Ejemplos Comunidad OpenProcessing
GLSL

El lenguaje de los shaders: corre en la GPU y genera efectos visuales en tiempo real. Va dentro de p5.js o Three.js. Muy potente para distorsionar imagen reactivamente.

Book of Shaders Shadertoy Glosario GLSL
Max / Pure Data

Programación por nodos y cables, sin texto. Para síntesis de audio y video en vivo. Pure Data es la versión gratuita y abierta de Max. Muy usado en instalaciones y performance.

Max Learn PD tutoriales PD comunidad
HTML + CSS

La base de cualquier página. Si el sistema corre en el navegador, entender HTML básico permite armarlo todo en un archivo sin instalar nada. El contenedor de todo lo demás.

MDN Learn CSS-Tricks CodePen

Registros georreferenciados de especies a lo largo del tiempo. Distribución, migración, aparición y desaparición con dimensión temporal explícita.

biodiversidad

Miles de datasets listos para usar. Clima, movimiento, sonido, series temporales. El más accesible para prototipado rápido dentro del día.

general

Cómo hablarle a la IA — consejos para el día

Prompts concretos para avanzar más rápido
y no gastar tiempo ni tokens de más.
Estrategia del día — cómo distribuir las 4 horas +

La mayoría de los equipos pierde tiempo yendo directo al código sin tener claro qué quieren hacer. Esta secuencia está pensada para llegar al final con algo que funcione y tenga peso conceptual.

bloque 1 — 10:00 a 10:30 · concepto

Antes de abrir el editor: ¿qué idea trabaja la pieza? ¿Qué pasa visualmente cuando los cuerpos se acercan, se tocan, se alejan? Definan una sola metáfora visual y comprométanse con ella.

bloque 2 — 10:30 a 12:00 · modelo + detección

Entrenar Teachable Machine con los cuerpos reales en el espacio real con la luz real. No en otro lado. Probar el modelo hasta que reconozca bien. Integrar con p5.js + ml5.

bloque 3 — 14:00 a 15:30 · animación + integración

Conectar la detección con la animación generativa. Que algo visible cambie cuando los cuerpos se mueven. No tiene que ser complejo: tiene que ser coherente con la idea del bloque 1.

bloque 4 — 15:30 a 16:00 · ajuste y presentación

Dejen de agregar features. Ajusten lo que ya funciona: velocidad, colores, sensibilidad. Ensayen la presentación de 4 minutos. El jurado evalúa concepto y experiencia, no cantidad de funciones.

No pegues el código entero de nuevo +

No pegues el código completo cada vez. Si ya lo escribió en el mismo chat, alcanza con señalar qué parte querés cambiar.

en vez de pegar todo, decile

"En el código que escribiste, modificá solo la función que detecta movimiento para que devuelva un valor entre 0 y 1 en lugar de true/false."

Pedile que te documente el código +

Antes de usar el código que te generó, pedile que te explique cada parte. Así sabés dónde tocar si algo no funciona.

prompt directo

"Documentá este código. Para cada función, explicá en una línea qué hace y qué pasa si cambio sus parámetros principales."

Pedile un mapa de parámetros +

Cuando el código funciona y querés explorar efectos sin entender todo, pedile una lista de qué tocar y qué cambia.

prompt directo

"¿Qué variables puedo modificar para cambiar velocidad, color, tamaño o intensidad del efecto? Dame el nombre de cada una, qué controla y qué rango tiene sentido."

Pedile sliders para explorar en vivo +

En lugar de editar el código cada vez que querés probar un valor, pedile que agregue controles directamente en la pantalla.

prompt directo

"Agregá sliders HTML al lado del canvas para controlar los parámetros principales en tiempo real. Que cada slider muestre su valor actual."

Construí en capas, no de una +

Pedir todo junto genera código difícil de debuggear. Pedí una parte, probala, y después seguís. Ahorra tiempo y tokens.

cómo empezar

"Primero hacé solo la parte que abre la cámara y muestra el feed en un canvas. Cuando eso funcione, te pido la detección de movimiento."

Pegá el error exacto de la consola +

No describas el error con tus palabras: copiá el mensaje exacto de la consola del navegador (F12 → Console). Eso le da contexto preciso.

prompt directo

"Este código me da el siguiente error en la consola: [pegá el error]. ¿Qué está mal y cómo lo corrijo?"

Pedile alternativas antes de comprometerte +

Si no sabés qué camino tomar, pedile opciones con distinto nivel de complejidad. Elegís la que se ajusta al tiempo que tienen.

prompt directo

"Dame 3 formas distintas de traducir la proximidad entre dos objetos en un cambio visual. Una muy simple, una intermedia y una más elaborada."

Pedile que explique antes de escribir +

Si te entrega código que no entendés, no lo uses todavía. Pedile que te diga qué va a hacer en lenguaje simple.

antes de ejecutar

"Antes de escribir el código, explicame en 3 pasos qué va a hacer y qué librerías va a usar. Después lo escribís."
